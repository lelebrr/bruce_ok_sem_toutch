# Caminho para a versão 2.0

O objetivo da versão 2.0 é migrar o framework para o Arduino-ESP32 v3.+, permitindo programar também para os chips ESP32-C5 (WiFi 5 GHz).

## Pontos negativos (trade-offs)

- Incompatibilidade com WireGuard
- Remover OpenHaystack (difícil de usar e com pouco ganho prático)
- Remover DPWO (transformar em algo rodando via JavaScript)

## Melhorias esperadas

- Uso de uma versão mais nova de C++
- Possibilidade de programar para ESP32-C5 (chip com WiFi de 5 GHz)

## Itens a serem alterados na 2.0

- [ ] Código comum do `InputHandler`, definido via estruturas de `#define`, ao invés de implementar funções separadas para cada dispositivo
- [ ] Funções comuns de PPM e FuelGauge
- [ ] Adicionar uma camada intermediária (MiddleClass) para TFT, permitindo usar mais bibliotecas (ArduinoGFX e Lovyan_GFX)
